我把 Rust 的学习拆成两条主线：**你能不能表达意图**，以及 **编译器能不能证明安全**。

## 你需要的 3 个概念

### 1) 所有权（Ownership）

- 值在同一时间只有一个 owner
- 离开作用域自动 drop

### 2) 借用（Borrowing）

- `&T`：共享借用（可多个）
- `&mut T`：可变借用（同一时间只能一个）

### 3) 生命周期（Lifetime）

生命周期不是“活多久”，而是“引用在类型系统里的约束”。大多数时候让编译器推导就够了。

## 一个常见误区

很多人把 borrow checker 当成限制，其实它是把你在 C/C++ 里“靠经验保证不踩雷”的事情，改成“靠编译器证明”。

## 结语

当你能用所有权/借用解释一段代码的行为时，你就已经跨过了最难的那道坎。
